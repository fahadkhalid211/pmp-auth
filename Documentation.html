📦 Installation

  1. Download the plugin ZIP Download pmp-2fa-authentication.zip from your CodeCanyon purchase page.
  2. Upload to WordPress Go to Plugins → Add New → Upload Plugin, choose the ZIP file and click Install Now. Alternatively, unzip and upload the pmp-2fa-authentication/ folder to /wp-content/plugins/ via FTP.
  3. Activate the plugin Go to Plugins → Installed Plugins and activate PMP 2FA Authentication.
  4. Configure the plugin Navigate to Settings → PMP 2FA Auth to set your preferred 2FA method and options.
⚠️
If you are upgrading from version 1.x, deactivate and delete the old version before installing 2.0.0. The plugin was completely rewritten and is not backward-compatible.

⚙️ Configuration

All settings live under Settings → PMP 2FA Auth. The plugin becomes active immediately after saving.

Choosing a 2FA Method

MethodDescription
Email OTP onlySends a one-time code to the user's WordPress email address. Works out of the box with no additional setup.
SMS only (Twilio)Sends a code via SMS. Requires Twilio credentials and a phone number on each user account. Falls back to email if Twilio is unreachable.
Both — user choosesDisplays an Email/SMS tab on the verification modal. Users can switch between methods. SMS tab is hidden if Twilio credentials are missing.

📱 SMS Setup (Twilio)

  1. Create a Twilio account Sign up at twilio.com. Free trial accounts work for testing.
  2. Get a Twilio phone number In the Twilio Console go to Phone Numbers → Manage → Buy a Number. Choose a number with SMS capability.
  3. Copy your credentials From your Twilio Console dashboard, copy the Account SID and Auth Token.
  4. Enter credentials in the plugin Go to Settings → PMP 2FA Auth → SMS Settings, paste your Account SID, Auth Token, and From Number (E.164 format: +12223334444), then save.
  5. Send a test SMS First, add your own phone number to your WordPress profile (see next section), then click Send Test SMS. A result message will appear next to the button.
✅
If the SMS send fails for any reason (Twilio error, no phone number, etc.), the plugin automatically falls back to sending the OTP by email. No login is ever blocked due to an SMS failure.

👤 User Phone Numbers

When SMS is enabled, users must save a phone number in their WordPress profile:

  1. Navigate to the profile Admin: Users → All Users → Edit (any user). User: Users → Profile.
  2. Enter the phone number In the Two-Factor Authentication section, enter the phone number in E.164 format: a + followed by the country code and number, e.g. +12223334444.
  3. Save the profile Click Update Profile. The field validates E.164 format and rejects invalid values silently.
ℹ️
If a user does not have a phone number on file and the method is set to SMS, the plugin falls back to email automatically.

🛡 Trusted Devices

When the Trust Device option is enabled, users can check "Trust this device for N days" on the 2FA modal. On subsequent logins from the same browser, 2FA is skipped for the configured period.

How it works

  • A random 40-character token is generated and stored in a hashed form in user-meta.
  • An HttpOnly, Secure cookie containing the plain token is set in the browser.
  • On login, the cookie is read, hashed, and compared against stored hashes. The user-agent is also fingerprinted as an extra check.
  • Expired entries are pruned automatically on every trust-device write.

Revoking trusted devices

Admins can revoke devices from Settings → PMP 2FA Auth → Revoke Trusted Devices. Users can revoke their own devices from their WordPress profile page.

🔌 Developer Hooks

Filter pmp2fa_login_redirect

Filters the URL users are redirected to after successfully completing 2FA.

add_filter( 'pmp2fa_login_redirect', function( $url, $user ) {
    // Redirect all users to a custom dashboard page.
    return home_url( '/my-dashboard/' );
}, 10, 2 );
ParameterTypeDescription
$urlstringThe redirect URL (defaults to PMP account page or admin).
$userWP_UserThe user who just logged in.

Action wp_login (standard WP)

Fired by the plugin immediately after auth-cookie is set, so other plugins receive the standard login action at the correct time.

💡
Define the constant PMP2FA_PASS to bypass 2FA programmatically (e.g. in custom login flows or CLI scripts). Use with care.

❓ FAQ

Does this work with the PMP frontend login shortcode?

Yes. The plugin intercepts authentication at the WordPress core level via the authenticate filter, so it works with all login methods — including the [pmpro_login] shortcode, the standard wp-login.php form, and any plugin that calls wp_signon().

Is the Twilio Auth Token stored securely?

Credentials are stored in the WordPress options table using standard WordPress sanitisation functions (sanitize_text_field). They receive the same protection as all other WordPress options (database-level access controls). Consider encrypting at-rest credentials at the server/database level for maximum security.

What if a user never receives the OTP email?

Check that WordPress can send email from your host. Install a transactional email plugin (e.g. WP Mail SMTP with SendGrid or Mailgun) to improve deliverability. The plugin's Send Test Email button is a quick diagnostic tool.

What happens if SMS fails?

If Twilio credentials are missing, the phone number is absent, or the Twilio API returns an error, the plugin silently falls back to sending the OTP by email. The user is never locked out.

Can I disable 2FA for admins?

Not via a built-in setting — 2FA applies to all users. If you need to bypass for specific roles, use the pmp2fa_login_redirect filter or define PMP2FA_PASS conditionally before login fires. A role-exclusion option is planned for a future release.

Does the plugin use PHP sessions?

No. PHP sessions require session_start() before any output, which causes "headers already sent" fatal errors on many hosts and themes. Instead, the plugin uses WordPress transients for server-side state and HttpOnly cookies for client-side state — a more reliable and WordPress-native approach.

How do I translate the plugin?

The plugin is fully translation-ready. All strings use the text domain pmp-2fa-authentication. Place .po / .mo files in /wp-content/languages/plugins/ or use a plugin like Loco Translate.

🔍 Troubleshooting

2FA modal does not appear after login

  • Make sure your PMP login page has wp_footer() in the theme template (standard themes do).
  • Check that the URL after failed login contains ?action=pmp2fa_pending. If not, another plugin may be interfering with the authenticate filter.
  • Verify the plugin is active and no PHP errors are in your error log.

OTP email is not being received

  • Use Settings → PMP 2FA Auth → Send Test Email to confirm basic email delivery works.
  • Check spam/junk folders.
  • Install WP Mail SMTP and configure a transactional email provider (SendGrid, Mailgun, Postmark, etc.).

"Session expired" error on the 2FA form

  • The pending-login transient (10-minute window) has expired. Return to the login page and sign in again.
  • If this happens immediately, check that your server clock is accurate (date in SSH). A clock skew of more than a few seconds will cause transients to expire incorrectly.
  • Ensure cookies are not being blocked by a caching plugin. Add pmp2fa_token to your cache plugin's cookie exclusion list.

SMS sends but the code is never received

  • Confirm the user's phone number is in E.164 format: + country code + number, e.g. +447700900123.
  • Check your Twilio account balance and whether the destination country is supported.
  • In your Twilio Console check the Message Logs for delivery status and any error codes.
